Seatbelt

Constitutional governance for agentic coding

npm GitHub v0.2.0

What is Seatbelt?

Seatbelt is a harness that enforces clean coding principles (SRP, small focused changes, no god files, etc.) while you work with AI coding agents. It is voluntary to adopt but strict once enabled.

The harness (not the model) owns review timing and correction loops. Violations trigger automatic restricted editing until the code passes the constitutional checks.

Installation

npm install -g @seanoseanohay/seatbelt

Then just run:

seatbelt # starts an interactive governed session
seatbelt "your task here" # one-shot mode

Key Features

Harness-Owned Timing

The system decides when reviews happen — not the model. No more self-review escape hatches.

Automatic Correction

When violations are detected, the agent is placed in restricted mode and must fix only the reported issues.

Configurable Constitution

Control thresholds and high-risk patterns via .seatbelt/config.json in your project.

Stable Sessions

Work in a persistent ~/.seatbelt/sessions/default worktree that survives across invocations.

Philosophy

Good engineering discipline should work like a seatbelt:

  • You can still drive without it (easy bypass)
  • Once you put it on, it actually protects you
  • The goal is to make using it the default, low-friction choice
